CSS 课程详细笔记（结合代码）

课程导入与背景介绍

主要使用的资料是一份 PDF 格式的 Flexbox 解析图纸。Flexbox（Flexible Box）布局是一种 CSS3 布局模型，用于在容器内有效地分配空间和排列项目。它解决了传统布局方式（如浮动布局和表格布局）的局限性，尤其适用于一维布局（单行或单列布局）。

Flexbox 布局的特点与基本属性

Flexbox 的特点：

  • 灵活性：Flexbox 可以动态调整容器内的项目大小和位置，以适应不同的屏幕尺寸和分辨率。
  • 简化布局代码：与传统的浮动布局相比，Flexbox 使复杂的布局更容易实现，减少了 CSS 代码的冗余。
  • 响应式设计：Flexbox 的属性能够帮助创建响应式布局，使得网页在不同设备上都有良好的显示效果。

1. Flexbox 基本属性和用法

1.1 创建 Flex 容器

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <link rel="stylesheet" href="styles.css">
  <title>Flexbox Demo</title>
</head>
<body>
  <div id="app">
    <div class="item">A</div>
    <div class="item">B</div>
    <div class="item">C</div>
    <div class="item">D</div>
    <div class="item">E</div>
    <div class="item">F</div>
  </div>
</body>
</html>

在上述 HTML 结构中，我们创建了一个包含 6 个子元素的 div 容器（id="app"）。这些子元素将被设置为 Flex 项目。

1.2 基础样式设置

/* styles.css */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

body {
  font-family: Arial, sans-serif;
  background-color: #f0f0f0;
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}

#app {
  display: flex;
  width: 80%;
  max-width: 1200px;
  background-color: #fff;
  border-radius: 8px;
  overflow: hidden;
  box-shadow: 0 4px 8px rgba(0, 0, 0, 0.1);
}

在上述 CSS 样式中，我们对所有元素进行了全局样式重置，设置了 margin 和 padding 为 0，并使用 box-sizing: border-box 以便包括内边距和边框在内的宽度和高度计算。

display: flex：将 #app 设为 Flex 容器，使其子元素（.item）成为 Flex 项目。

1.3 Flex 容器的方向与对齐方式

#app {
  display: flex; /* 定义为 Flex 容器 */
  flex-direction: row; /* 主轴方向为水平方向（默认值） */
  justify-content: flex-start; /* 主轴方向上的对齐方式 */
  align-items: flex-start; /* 交叉轴方向上的对齐方式 */
}

flex-direction：定义主轴的方向。row 是默认值，表示项目沿水平方向从左到右排列。column 则表示沿垂直方向从上到下排列。

justify-content：定义项目在主轴方向上的对齐方式。常用值包括：

  • flex-start：项目从主轴起点对齐（默认值）。
  • flex-end：项目从主轴终点对齐。
  • center：项目居中对齐。
  • space-between：项目均匀分布，第一个项目在起点，最后一个项目在终点。
  • space-around：项目均匀分布，每个项目两侧的间距相等。

align-items：定义项目在交叉轴上的对齐方式。常用值包括：

  • flex-start：项目从交叉轴起点对齐。
  • flex-end：项目从交叉轴终点对齐。
  • center：项目在交叉轴上居中对齐。
  • baseline：项目的基线对齐。
  • stretch：项目在交叉轴上拉伸以填满容器（默认值）。

1.4 项目间距与换行

#app {
  display: flex;
  flex-wrap: wrap; /* 允许换行 */
  gap: 20px; /* 设置项目间距 */
}

flex-wrap：定义项目是否在主轴方向上换行。常用值包括：

  • nowrap：不换行，所有项目都在一行显示（默认值）。
  • wrap：如果项目超出容器宽度，项目将换行显示。
  • wrap-reverse：与 wrap 类似，但换行的方向相反。

gap：定义项目之间的间距。

1.5 项目的增长、收缩与基础大小

.item {
  flex-grow: 1; /* 项目将会平分容器的剩余空间 */
  flex-shrink: 1; /* 项目可以缩小以适应容器 */
  flex-basis: 200px; /* 项目的基础大小 */
}

flex-grow：定义项目在主轴方向上如何扩展空间。默认值为 0，表示项目不会扩展。如果设置为 1，项目将在剩余空间中等比例扩展。

flex-shrink：定义项目在主轴方向上如何收缩空间。默认值为 1，表示项目会收缩以适应容器。如果设置为 0，项目将不会收缩。

flex-basis：定义项目的初始主轴大小，可以使用任何 CSS 长度单位（例如 px、em、rem 等）。flex-basis 会覆盖项目的宽度或高度（取决于 flex-direction），并作为 flex-grow 和 flex-shrink 的计算基准。

1.6 单个项目的对齐与顺序调整

.item:nth-child(2) {
align-self: center; /* 单独设置第二个项目在交叉轴上的对齐方式 */
order: 1; /* 将第二个项目设置为第一个 */
}

align-self：允许单个 Flex 项目的对齐方式不同于其他项目。默认值为 auto，表示继承 align-items 的值。

order：定义项目的显示顺序。默认值为 0，可以设置正负值，数值越小的项目显示越靠前。

Flexbox 布局的综合应用

通过以上的代码示例和解释，学生可以理解如何使用 Flexbox 属性实现各种布局效果。下面是一些综合应用的示例：

2.1 创建一个简单的导航栏

<nav>
<ul class="navbar">
<li>Home</li>
<li>About</li>
<li>Services</li>
<li>Contact</li>
</ul>
</nav>

.navbar {
display: flex;
justify-content: space-between; /* 项目均匀分布，首尾项目贴边 */
background-color: #333;
padding: 1rem;
list-style: none;
}

.navbar li {
color: white;
font-size: 1.2rem;
}

在此示例中，Flexbox 被用来创建一个简单的水平导航栏，菜单项之间均匀分布，并在首尾各贴边。

2.2 实现一个自适应卡片布局

<div class="card-container">
<div class="card">Card 1</div>
<div class="card">Card 2</div>
<div class="card">Card 3</div>
<div class="card">Card 4</div>
</div>

.card-container {
display: flex;
flex-wrap: wrap; /* 项目超出容器宽度时换行 */
gap: 20px; /* 卡片之间的间距 */
justify-content: center; /* 卡片居中对齐 */
}

.card {
flex: 1 1 200px; /* 设置卡片的增长、收缩和基础大小 */
background-color: #f9f9f9;
border: 1px solid #ddd;
border-radius: 8px;
padding: 1rem;
box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
}

flex-wrap: wrap：允许卡片在容器宽度不足时换行，保证布局的弹性。

gap: 20px：设置卡片之间的间距。

flex: 1 1 200px：结合了 flex-grow、flex-shrink 和 flex-basis 属性，使得每个卡片的基础大小为 200px，可以根据需要增长或收缩。

2.3 创建一个响应式的侧边栏布局

<div class="layout">
<aside class="sidebar">Sidebar</aside>
<main class="content">
<h1>Main Content</h1>
<p>This is the main content area.</p>
</main>
</div>

.layout {
display: flex;
flex-wrap: nowrap; /* 禁止换行，确保侧边栏和内容区域在一行 */
height: 100vh; /* 全屏高度 */
}

.sidebar {
flex: 0 0 250px; /* 固定宽度的侧边栏 */
background-color: #333;
color: white;
padding: 1rem;
}

.content {
flex: 1; /* 自适应宽度的内容区域 */
padding: 1rem;
background-color: #f4f4f4;
}

flex-wrap: nowrap：禁用换行，确保侧边栏和内容区域保持在同一行。

.sidebar：设置侧边栏的宽度为 250px，并不允许扩展或收缩。

.content：内容区域设置为 flex: 1，表示它将占据所有剩余的可用空间。

2.4 制作一个复杂的网格布局

<div class="grid">
<div class="grid-item">Item 1</div>
<div class="grid-item">Item 2</div>
<div class="grid-item">Item 3</div>
<div class="grid-item">Item 4</div>
<div class="grid-item">Item 5</div>
<div class="grid-item">Item 6</div>
</div>

.grid {
display: flex;
flex-wrap: wrap;
gap: 10px;
}

.grid-item {
flex: 1 1 calc(33.333% - 20px); /* 三等分宽度减去间隙 */
background-color: #ececec;
padding: 20px;
box-sizing: border-box;
}

flex: 1 1 calc(33.333% - 20px)：每个网格项的宽度设为容器宽度的三分之一，减去 gap 的一部分。

calc：CSS 函数，用于动态计算长度值，支持基本的数学运算。

flex-wrap: wrap：项目换行，确保在不同屏幕尺寸下的自适应。

2.5 创建一个具有 sticky footer 的页面布局

<div class="page-container">
<header class="header">Header</header>
<main class="main-content">Main Content</main>
<footer class="footer">Footer</footer>
</div>

html, body {
margin: 0;
padding: 0;
height: 100%;
}

.page-container {
display: flex;
flex-direction: column;
min-height: 100vh;
}

.header {
background-color: #4CAF50;
color: white;
text-align: center;
padding: 1rem;
}

.main-content {
flex: 1; /* 自适应高度 */
padding: 2rem;
background-color: #f0f0f0;
}

.footer {
background-color: #333;
color: white;
text-align: center;
padding: 1rem;
}

flex-direction: column：设置 Flex 容器的主轴为垂直方向（列）。

min-height: 100vh：容器的最小高度为 100% 的视口高度，确保页脚固定在底部。

flex: 1：内容区域自适应高度，占据所有剩余空间，使页脚始终在页面底部。

总结

通过这些综合应用示例，学生可以更好地理解如何利用 Flexbox 属性创建各种复杂而灵活的布局。Flexbox 布局模型提供了强大的功能，能够简化 CSS 代码并提高网页的响应性和用户体验。熟练掌握这些技巧，能够让开发者更高效地构建现代化的 Web 页面。
