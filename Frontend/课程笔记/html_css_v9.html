1. 项目设置与安装

1.1 下载 src 压缩包

在课程中提到的链接或指定的网络存储中，下载名为 src 的压缩包，大小超过 80MB。这个压缩包包含项目所需的初始文件和资源，如图标、图片、JavaScript 文件等。下载完成后，将压缩包解压到项目的根目录下，确保所有文件和文件夹都正确解压出来。

1.2 安装开发环境

使用 Yarn 初始化项目：

  1. 打开 VS Code，确保你的计算机上已经安装了 Node.js 和 Yarn。
  2. 使用 Yarn 进行项目初始化。在终端中输入以下命令来创建一个新的项目：
yarn create project-a1

这个命令会启动一个 Yarn 创建向导，引导你选择项目路径和 JavaScript 环境。你可以选择 JavaScript 或 TypeScript，取决于项目的需求。

安装必要的依赖：

使用 Yarn 安装必要的依赖包，例如 Sass，用于编写和处理样式文件：

yarn add sass -D

-D 选项表示这是一个开发依赖项（devDependencies），仅在开发环境中使用。

2. 项目结构初始化

2.1 清理和设置项目结构

删除不必要的文件和资源：

打开项目文件夹，删除不需要的文件，如 public 目录下的 wait.svg 和 container.js 文件。这些文件在当前项目中没有用处，删除它们可以保持项目的简洁和整洁。

修改 index.html：

打开 index.html 文件，更新页面的基本结构和内容：

  • 删除对已删除资源的引用，例如 wait.svg。
  • 修改 <title> 标签为项目的名称，例如 Project A1，以便在浏览器标签上显示正确的项目名称。

更新样式文件：

将原来的 style.css 文件重命名为 style.scss，以使用 Sass 的高级功能。清空 style.scss 文件的内容，以便重新设置和定义样式。

2.2 设置基本项目框架

创建基础文件和文件夹结构：

├── src/
│   ├── icons/
│   ├── images/
│   ├── scripts/
├── public/
├── styles/
│   ├── style.scss
├── index.html

src 文件夹包含所有的源代码和资源文件，public 文件夹用于存放公开访问的文件，styles 文件夹用于存放样式文件。

定义页面的基本结构：

在 index.html 中，使用 HTML5 语义化标签来定义页面的结构，包括 header、main、footer 等：

<header>
  <!-- 头部内容 -->
</header>
<main>
  <!-- 主体内容 -->
</main>
<footer>
  <!-- 底部内容 -->
</footer>

3. 项目打包与分享

3.1 打包项目

打包项目文件：

在打包项目时，确保不将 node_modules 文件夹包含在压缩包内，因为该文件夹通常非常大且可以通过运行 yarn install 来重新生成。

使用以下命令将项目文件打包为 .zip 文件：

zip -r project-a1.zip . -x "node_modules/*"

该命令会递归压缩当前目录下的所有文件和文件夹，但会排除 node_modules 文件夹。

3.2 分享项目

分享打包好的项目：

将打包好的 .zip 文件上传到课程指定的文件共享平台或直接分享给需要的同学。这样可以确保大家使用相同的项目结构和依赖版本。

版本兼容性注意事项：

老师特别强调了版本兼容性的问题。由于框架和依赖库会随着时间更新，如果将某个版本的框架打包使用，可能会与未来的版本不兼容。因此，建议每次使用框架时都重新安装依赖，确保使用最新兼容的版本。

4. 代码结构和样式调整

4.1 定义项目基本结构

使用 HTML5 语义化标签：

使用 HTML5 的 header、main、footer 等标签来定义页面的布局结构，这样做有助于增强页面的语义性和可读性：

<header>
  <nav>
    <!-- 导航栏内容 -->
  </nav>
</header>
<main>
  <section>
    <!-- 主要内容部分 -->
  </section>
</main>
<footer>
  <!-- 页脚内容 -->
</footer>

使用 CSS 设置 SVG 图标：

在样式文件 style.scss 中，使用 background 属性来设置 SVG 图标的样式，并使用 filter 属性来调整图标颜色：

.menu-logo {
background: url('src/icons/menu.svg') no-repeat center;
width: 24px;
height: 24px;
filter: invert(1); /* 将黑色图标反转为白色 */
}

通过这种方式，可以轻松地调整 SVG 图标的颜色，而不需要修改图标本身。

4.3 使用 Flexbox 进行布局

使用 Flexbox 设置布局：

在 style.scss 中，使用 Flexbox 布局来管理页面元素的排列和对齐方式：

header, footer {
display: flex;
justify-content: space-between;
align-items: center;
}

main {
display: flex;
flex-direction: column;
flex-grow: 1;
}

Flexbox 提供了强大的布局功能，可以方便地进行水平和垂直方向上的对齐和分布。

5. 功能实现与页面优化

5.1 添加动画效果

使用 CSS 动画属性：

使用 CSS animation 属性为页面元素添加动态效果，例如淡入效果。可以使用 @keyframes 定义动画关键帧：

@keyframes fadeIn {
from { opacity: 0; }
to { opacity: 1; }
}

.fade-in {
animation: fadeIn 0.5s ease-in-out;
}

这种动画可以用于页面加载时的元素渐入效果，提升用户体验。

5.2 实现响应式设计

使用媒体查询实现响应式设计：

使用 @media 查询来设置不同屏幕尺寸下的样式，确保页面在各种设备上的显示效果：

@media (max-width: 768px) {
.menu-logo {
width: 20px;
height: 20px;
}
}

响应式设计可以确保页面在不同大小的屏幕上都能良好显示，提供一致的用户体验。

5.3 添加按钮效果

为按钮添加 hover 和 active 状态：

在样式文件中，为按钮元素设置 hover 和 active 状态的样式，以增强交互效果：

button {
background-color: $main-color;
color: #fff;
padding: 10px 20px;
border-radius: 5px;
border: none;
cursor: pointer;
transition: background-color 0.3s ease;
}

button:hover {
background-color: darken($main-color, 10%);
}

button:active {
background-color: lighten($main-color, 10%);
}

使用 transition 属性，可以平滑地过渡按钮颜色的变化。

6. 字体设置与兼容性调整

6.1 加载 Google 字体

引入 Google 字体：

使用 Google Fonts 为页面引入所需字体，在 style.scss 文件中进行字体的引入和设置：

@import url('https://fonts.googleapis.com/css2?family=Poppins:wght@400;600&display=swap');

body {
font-family: 'Poppins', sans-serif;
}

这可以确保页面的字体在各种浏览器和设备上都能正确显示。

6.2 设置字体与行高

使用 em 单位设置行高：

为什么使用 em 单位：

em 是相对单位，它基于当前元素的字体大小来计算。使用 em 单位设置行高（line-height），可以确保文本的行高相对于字体大小进行缩放，从而在不同的字体大小下保持一致的比例。

例如，line-height: 1.5em; 意味着行高是字体大小的 1.5 倍。无论字体大小如何改变，行高都将相应调整，确保文本的可读性和视觉美观。

如何在项目中应用 em 单位：

在 style.scss 中，为文本元素如 h1、h2、p 等设置字体大小和行高：

body {
font-size: 16px; /* 基础字体大小 */
}

h1 {
font-size: 2em; /* 32px */
line-height: 1.2em; /* 1.2 倍字体大小，行高为 38.4px */
}

p {
font-size: 1em; /* 16px */
line-height: 1.5em; /* 1.5 倍字体大小，行高为 24px */
}

这种做法使得字体和行高能够根据基准字体大小灵活调整，从而保持布局的一致性。

设置响应式字体和行高

使用媒体查询进行响应式调整：

为了确保字体和行高在不同设备上的可读性，使用媒体查询（@media）在不同屏幕尺寸下调整字体大小和行高：

@media (max-width: 768px) {
body {
font-size: 14px; /* 较小屏幕下调整基础字体大小 */
}

h1 {
font-size: 1.8em; /* 较小的 h1 字体 */
line-height: 1.3em; /* 较大的行高确保可读性 */
}

p {
font-size: 0.9em; /* 较小的 p 字体 */
line-height: 1.6em; /* 较大的行高确保文本间距 */
}
}

这种方式可以确保在小屏幕设备（如手机）上字体不会显得过大或过小，同时行高也适应较窄的屏幕宽度。

7. 课程作业与练习

7.1 作业要求

为其他部分添加动画效果：

要求学生为项目的其他部分（如 header、footer）添加动画效果。可以使用类似的 @keyframes 动画定义来实现不同的动画效果。

例如，可以为 header 添加淡入淡出效果，为 footer 添加滑入效果：

@keyframes slideIn {
from { transform: translateY(100%); }
to { transform: translateY(0); }
}

footer {
animation: slideIn 0.8s ease-in-out;
}

使用媒体查询实现响应式设计：

要求学生使用 @media 查询为不同设备设置响应式样式，确保页面在不同屏幕尺寸下的显示效果。

学生应根据设备类型（如手机、平板、桌面）调整字体大小、行高、布局等属性。

7.2 额外任务

探索更多 CSS 动画和样式设置：

鼓励学生探索更多 CSS 动画和样式设置，增强页面的视觉效果和用户体验。

例如，研究不同的动画曲线（如 ease-in、ease-out、linear 等）如何影响动画效果，尝试不同的动画持续时间和延迟时间。

掌握布局和响应式设计方法：

鼓励学生深入理解 Flexbox 和 Grid 布局，了解它们在不同场景下的使用方法。

掌握如何使用 @media 查询和相对单位（如 em、rem）来创建响应式设计，确保页面在各种设备上的一致性和适应性。
